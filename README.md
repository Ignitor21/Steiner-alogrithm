# Реализация алгоритма Iterated 1-Steiner (I1S)

## 1. Запуск программы

**Базовый алгоритм:**

```bash
python My-Steiner.py <test_name>.json
```

**Модифицированный алгоритм:**

```bash
python My-Steiner.py -m <test_name>.json
```

Выходной файл `<test_name>_out.json` создаётся в текущей директории
(там, где запущена программа). Дополнительно в корень
JSON сохраняются поля `algorithm`, `total_length`, `elapsed_seconds`.

Пример:

```bash
python My-Steiner.py test_example.json
# -> ./test_example_out.json
```

Программа гарантирует, что полученное дерево:

- содержит ровно `N − 1` рёбер для `N` узлов;
- является связным;
- все точки Штейнера имеют степень `≥ 3` (точки меньшей степени
  удаляются на постобработке).

Дополнительно в проекте есть вспомогательные скрипты:

- `run_benchmarks.py` — запускает оба алгоритма на всех тестах
  бенчмарка и формирует сводные таблицы в `results/`.
- `validate_outputs.py` — проверяет корректность всех выходных
  файлов (число рёбер, связность, степень точек Штейнера,
  согласованность заявленной длины с суммой длин рёбер).
- `plot_example.py` — рисует результат в PNG для наглядности.

## 2. Модификация Batched I1S

Идея модификации: на одной итерации добавлять сразу несколько точек
Штейнера, а не одну. Если умело выбирать такой батч, то общее число
дорогих итераций можно сократить.

**Алгоритм модификации:**

1. Как и в базовом варианте, считаем индивидуальные выигрыши `Δ_c`
   для всех кандидатов `c` из сетки Ханана.
2. Сортируем кандидатов по убыванию `Δ_c`.
3. Жадно выбираем из них «пакет» точек по правилу: у каждого нового
   кандидата в пакете хотя бы одна координата (`x` или `y`) должна
   отличаться от уже выбранных в этом же пакете. Смысл правила — взять
   точки, воздействующие на разные участки дерева, чтобы их
   индивидуальные выигрыши почти не конфликтовали между собой.
4. Добавляем весь пакет в `P` и пересчитываем RMST один раз (а не
   по разу на каждую точку, как в базовом).
5. Eсли после пакетного добавления реальная длина RMST
   всё-таки не уменьшилась (редкий случай, когда независимые оценки
   обманули), откатываемся и добавляем только одного лучшего
   кандидата — как в базовом I1S. Это гарантирует, что Batched I1S
   никогда не увеличивает длину дерева.

Случаи применения каждого из алгоритмов:
- базовый I1S — когда приоритетно качество решения;
- Batched I1S — когда нужно обрабатывать много входов или более
  крупные множества терминалов.
